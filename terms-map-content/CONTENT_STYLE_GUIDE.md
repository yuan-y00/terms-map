# 术语地图内容风格指南

## 写作目标

帮助读者把遇到的工程词、工程对象和相关知识连起来，不写完整教材，也不写开发口号。

推荐结构：

```text
工程对象或术语
→ 它是什么意思
→ 为什么容易卡住
→ 它牵到哪些知识
→ 最低掌握目标
```

## 语气

- 说人话。
- 少用抽象分类。
- 不写招生宣传。
- 不把资源包装成完整学习计划。
- 能用真实工程例子解释，就不要只给定义。

## 资源卡片

资源必须说明：

- 用户遇到什么词或问题时该看它。
- 它能帮用户补上哪类知识。
- 它对应术语表里的哪些关键词。
- 它的最低使用目标是什么。

## 不做

- 不维护德国申请、职业、居留路线。
- 不维护泛泛资源入口。
- 不搬运外部内容。
- 不编造来源。
- 不把未经核验的时效性信息写成事实。
